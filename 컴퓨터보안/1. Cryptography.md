**Cryptography란 악의적인 공격자에 대해 안전한 통신을 가능하게 하기 위한 기술이다.**

Cryptography에서 **cryptographic primitives**는 컴퓨터 보안 시스템에서 사용되는 기본적인 저수준 알고리즘으로, 다양한 보안 기능을 구현하기 위한 핵심 구성 요소이다. 이러한 primitive들은 기밀성, 무결성, 인증과 같은 개별적인 기능을 수행하며, 독립적으로 사용되기보다는 상위 구조를 구성하는 데 활용된다.

한편, **cryptographic protocol**은 이러한 primitive들을 조합하여 실제 보안 목적을 수행하도록 설계된 절차 또는 규칙이다. 즉, 여러 암호학적 기법을 일정한 순서로 적용하여 통신 보안, 인증, 키 교환과 같은 보안 기능을 구현한다.

결과적으로, cryptographic primitives가 개별적인 기능을 수행하는 기본 도구라면, cryptographic protocol은 이들을 조합하여 실제 보안 시스템을 구성하는 상위 개념이라고 할 수 있다.

---
암호 알고리즘과 프로토콜은 크게 네 가지 영역으로 분류할 수 있다

- 대칭키 암호
- 비대칭키 암호
- 데이터 무결성 알고리즘
- 인증 프로토콜이다

대칭키 암호는 하나의 동일한 키를 사용하여 암호화와 복호화를 수행하며, 효율적인 기밀성을 제공한다. 반면 비대칭키 암호는 공개키와 개인키의 쌍을 사용하여 키 분배 문제를 해결하고 보다 안전한 통신을 가능하게 한다.

데이터 무결성 알고리즘은 해시 함수나 메시지 인증 코드 등을 이용하여 데이터가 전송 과정에서 변조되지 않았음을 보장한다. 인증 프로토콜은 통신에 참여하는 개체의 신원을 확인하고 신뢰 관계를 형성하는 역할을 한다.

# CIA Triad
보안의 주요 목표는 기밀성, 무결성, 가용성, 인증성, 책임성으로 구분할 수 있다.

- **기밀성(Confidentiality)** 은 정보에 대한 접근과 공개를 허가된 사용자로 제한하여, 비인가자가 정보를 열람하지 못하도록 보호하는 것이다.  
- **무결성(Integrity)** 은 정보가 부적절하게 변경되거나 파괴되지 않도록 보호하여, 데이터의 정확성과 일관성을 유지하는 것을 의미한다.  
- **가용성(Availability)** 은 필요할 때 적절한 시간에 신뢰할 수 있는 방식으로 정보에 접근하고 사용할 수 있도록 보장하는 것이다.
- **인증성(Authenticity)** 은 정보나 사용자가 진짜임을 확인하고 신뢰할 수 있는 상태를 의미하며, 사용자가 주장하는 신원이 실제와 일치하는지, 시스템에 입력되는 데이터가 신뢰할 수 있는 출처로부터 온 것인지를 검증하는 것이다.  
- **책임성(Accountability)** 은 시스템 내에서 수행된 행위가 특정 개체에게 고유하게 추적될 수 있도록 하는 보안 목표로, 보안 사고 분석이나 분쟁 해결을 위해 시스템은 활동 기록을 유지해야 한다.

---
# Security
**사이버 보안(Cyber Security)** 은 사이버 공간과 조직 및 사용자 자산을 보호하기 위해 사용되는 도구, 정책, 보안 개념, 보호 장치, 지침, 위험 관리 방법, 대응 활동, 교육, 모범 사례, 보증 및 기술 등을 모두 포함하는 포괄적인 개념이다.

**정보 보안(Information Security)** 은 정보의 기밀성, 무결성, 가용성을 유지하는 것을 의미한다.

**네트워크 보안(Network Security)**은 네트워크와 그 서비스가 비인가된 수정, 파괴, 또는 정보 유출로부터 보호되도록 하는 것을 의미하며, 동시에 네트워크가 중요한 기능을 정상적으로 수행하고 유해한 부작용 없이 동작하도록 보장하는 것이다.

이 외에도 보안은 여러 영역으로 나뉘는데, **소프트웨어 보안(Software Security)** 은 소프트웨어의 취약점을 최소화하고 안전하게 동작하도록 하는 것을 의미하며, **웹 보안(Web Security)** 은 웹 애플리케이션과 웹 서비스에서 발생할 수 있는 공격을 방지하는 데 초점을 둔다. 또한 **시스템 보안(System Security)** 은 운영체제와 전체 시스템 자원을 보호하여 안정적이고 안전한 환경을 유지하는 것을 의미한다.

---
# Threats and Attacks
**위협(Threat)** 은 보안을 침해하고 피해를 발생시킬 수 있는 가능성을 의미하며, 특정한 상황, 능력, 행위 또는 사건에 의해 보안이 위반될 수 있는 잠재적 위험이다. 즉, 위협은 취약점을 악용할 수 있는 가능한 위험 요소라고 할 수 있다.

**공격(Attack)** 은  이러한 위협이 실제로 실행된 형태로, 시스템의 보안을 침해하기 위한 지능적이고 의도적인 행위이다. 이는 보안 서비스를 우회하고 시스템의 보안 정책을 위반하기 위해 특정한 방법이나 기술을 사용하는 것을 의미한다.

---
# OSI Security Architecture
- **보안 공격(Security Attack)** 은 정보의 보안을 침해하는 모든 행위를 의미한다.

- **보안 메커니즘(Security Mechanism)** 은 이러한 보안 공격을 탐지하고, 예방하거나, 발생 이후에 복구하기 위해 설계된 절차나 방법을 의미한다.

- **보안 서비스(Security Service)** 는 시스템의 보안을 향상시키기 위해 제공되는 처리 서비스로, 보안 공격에 대응하기 위한 목적을 가지며 하나 이상의 보안 메커니즘을 활용하여 구현된다.

---
# Security Attack
**보안 공격(Security Attack)** 은 정보 보안을 침해하는 행위를 의미하며, X.800과 RFC 4949에서는 이를 크게 **passive attack** 과 **active attack** 으로 분류한다.

![](../images/Pasted%20image%2020260418161509.png)

**수동적 공격(Passive Attack)** 은 시스템 자원에는 영향을 주지 않으면서 정보를 획득하거나 이용하려는 공격이다. 데이터의 변경이 발생하지 않기 때문에 탐지가 어렵다는 특징이 있다. 대표적으로 통신 내용을 도청하거나 전송 흐름을 감시하여 메시지 내용을 획득하거나 트래픽 패턴을 분석하는 공격이 이에 해당한다.

반면, **능동적 공격(Active Attack)** 은 시스템 자원을 변경하거나 정상적인 동작에 영향을 미치려는 공격이다. 데이터의 일부를 수정하거나 위조된 데이터를 생성하는 등 직접적인 변조가 발생하며, 다양한 물리적·소프트웨어적·네트워크 취약점을 이용하기 때문에 예방이 어렵다.

능동적 공격의 주요 유형은 다음과 같다.

- **위장(Masquerade)**: 한 개체가 다른 개체인 것처럼 가장하는 공격
- **재전송(Replay)**: 이전에 수집한 데이터를 다시 전송하여 비정상적인 효과를 유도하는 공격
- **데이터 변조(Data Modification)**: 메시지의 일부를 변경하거나 순서를 바꾸는 등의 방식으로 내용을 조작하는 공격
- **서비스 거부(DoS, Denial of Service)**: 정상적인 통신이나 시스템 사용을 방해하는 공격

# Security Services
**데이터 기밀성(Data Confidentiality)** 은 전송되는 데이터가 수동적 공격으로부터 보호되도록 하는 것을 의미한다. 또한 트래픽 분석으로부터도 보호하여, 통신의 출발지와 목적지, 전송 빈도, 길이 등과 같은 정보가 노출되지 않도록 한다.

**데이터 무결성(Data Integrity)** 은 메시지가 전송된 그대로 수신되도록 보장하는 것으로, 데이터의 중복, 삽입, 수정, 순서 변경, 재전송 등이 발생하지 않았음을 확인하는 것이다.

**부인 방지(Nonrepudiation)** 는 송신자나 수신자가 메시지 전송 사실을 부인하지 못하도록 하는 것을 의미한다. 즉, 수신자는 특정 송신자가 실제로 메시지를 보냈음을 증명할 수 있고, 송신자 또한 수신자가 해당 메시지를 받았음을 증명할 수 있다.

**가용성(Availability)** 은 시스템이 필요할 때 정상적으로 접근되고 사용될 수 있도록 보호하는 것을 의미하며, 특히 서비스 거부 공격(DoS)으로부터 시스템을 보호하는 데 초점을 둔다.

# Security Mechanisms
**암호 알고리즘(Cryptographic Algorithms)** 은 데이터를 보호하기 위한 다양한 암호 기술을 의미하며, 암호화(encipherment), 암호학적 기본 요소(cryptographic primitives), 암호화 알고리즘, 해시 알고리즘 등이 포함된다.

**데이터 무결성(Data Integrity)** 은 데이터 단위가 전송 과정에서 변조되지 않았음을 보장하기 위한 다양한 메커니즘을 포함한다.

**디지털 서명(Digital Signature)** 은 데이터의 출처와 무결성을 증명하며, 위조를 방지하는 역할을 한다.

**인증 교환(Authentication Exchange)** 은 정보 교환을 통해 개체의 신원을 확인하기 위한 메커니즘이다.

**트래픽 패딩(Traffic Padding)** 은 데이터 스트림의 빈 공간에 임의의 비트를 삽입하여 트래픽 분석을 어렵게 만드는 기법이다.

**공증(Notarization)** 은 신뢰할 수 있는 제3자를 이용하여 데이터 교환의 특정 속성(예: 시간, 무결성, 송수신 사실 등)을 보증하는 것이다.

# Kerckhoffs’s Principle
**커크호프의 원리(Kerckhoffs’s Principle)** 는 암호 시스템의 모든 세부 사항이 공개되더라도, 오직 비밀 키만 안전하게 유지된다면 시스템은 여전히 안전해야 한다는 원칙이다.

즉, 암호 알고리즘 자체는 비밀일 필요가 없으며, 적에게 알려지더라도 문제가 없어야 한다. 따라서 암호 시스템의 안전성은 알고리즘의 비밀성에 의존해서는 안 되고, 오직 비밀 키의 보안에만 의존해야 한다.
