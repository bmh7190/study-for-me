Boyer-Moore-Horspool 알고리즘은 문자열 매칭 알고리즘의 한 종류로, 찾고자 하는 패턴 `P`를 텍스트 `T` 위에 놓고 **패턴의 가장 오른쪽 문자부터 비교**한다. 

일반적인 brute-force 방식은 패턴의 앞 문자부터 비교하지만, BMH는 뒤에서부터 비교하기 때문에 불일치가 발생했을 때 패턴을 여러 칸 건너뛰며 이동할 수 있다. 강의자료에서도 BMH는 패턴의 가장 뒷 문자부터 탐색하고, 불일치가 발생하면 규칙에 따라 패턴을 오른쪽으로 이동한다고 설명한다.

이때 패턴을 얼마나 이동시킬지를 결정하기 위해 **last occurrence 함수 `L`** 을 사용한다. `L(σ)`는 알파벳 문자 `σ`가 패턴 `P` 안에서 **가장 마지막으로 등장하는 인덱스**를 의미한다. 만약 해당 문자가 패턴 안에 없다면 `-1`을 반환한다. 

![](../images/Pasted%20image%2020260520153435.png)

예를 들어 패턴이 `abcada`라면, `a`의 마지막 등장 위치는 5, `b`는 1, `c`는 2, `d`는 4이고, 나머지 문자는 `-1`이 된다.

# 의사 코드

```c
Algorithm BoyerMooreMatch(T, P, Σ)

L ← lastOccurrenceFunction(P, Σ)

i ← m - 1
j ← m - 1

repeat
    if T[i] = P[j] then
        if j = 0 then
            return i
        else
            i ← i - 1
            j ← j - 1
    else
        l ← L[T[i]]
        i ← i + m - min(j, 1 + l)
        j ← m - 1

until i > n - 1

return -1
```

맞은 경우에는 패턴의 끝에서부터 비교하는 방식이므로, `i`와 `j`를 하나씩 왼쪽으로 이동시키면서 이전 문자들도 계속 일치하는지 검증한다.

이 과정을 반복하다가 `j = 0`까지 도달하면 패턴 전체가 일치한 것이므로 매칭에 성공한 것이다.

반대로 문자가 틀린 경우에는 현재 텍스트 문자 `T[i]`를 기준으로 패턴을 얼마나 이동할지 결정한다. 이때 `l = L[T[i]]`라고 하면, `l`은 현재 텍스트 문자 `T[i]`가 패턴 안에서 마지막으로 등장하는 위치를 의미한다.

불일치가 발생했을 때는 크게 두 가지 경우로 나누어 생각할 수 있다.

### Case 1. 같은 문자가 현재 비교 위치보다 오른쪽에 있는 경우
패턴 안에서 `T[i]`와 같은 문자의 마지막 등장 위치가 현재 비교 중인 패턴 위치 `j`보다 오른쪽에 있는 경우이다.

![](../images/Pasted%20image%2020260520155919.png)
이 경우에는 해당 문자에 맞춰 패턴을 이동해도 현재 위치에서 유효한 매칭 후보를 만들기 어렵다. 따라서 패턴을 최소 한 칸 오른쪽으로 이동시킨다.

즉, 현재 위치에서는 더 이상 매칭 가능성이 없으므로 한 칸 이동한 뒤 다시 패턴의 마지막 문자부터 비교한다.

### Case 2. 같은 문자가 현재 비교 위치보다 왼쪽에 있는 경우
패턴 안에서 `T[i]`와 같은 문자의 마지막 등장 위치가 현재 비교 중인 패턴 위치 `j`보다 왼쪽에 있는 경우이다.

![](../images/Pasted%20image%2020260520155804.png)

이 경우에는 현재 텍스트 문자 `T[i]`와 패턴 안의 같은 문자가 서로 맞도록 패턴을 오른쪽으로 이동시킨다.

즉, 단순히 한 칸만 이동하는 것이 아니라, 패턴 안에서 같은 문자가 등장한 위치를 현재 텍스트 문자 위치에 맞춰준다. 이렇게 하면 불필요한 비교를 건너뛸 수 있다.

# 시간 복잡도
최악의 경우를 생각해보면, 패턴의 맨 뒤에서부터 비교를 시작했을 때 대부분의 문자가 일치하다가 맨 앞 문자에서만 불일치가 발생하는 경우이다.

길이가 `n`인 텍스트에서 길이가 `m`인 패턴을 찾는다고 생각해보자

BMH 알고리즘은 불일치가 발생하면 last occurrence 함수를 이용해 패턴을 이동시키는데, 이 경우에는 패턴이 한 칸 정도만 이동하게 된다. 따라서 한 번의 위치에서 최대 `m`개의 문자를 비교하고, 이런 비교가 텍스트 전체에 대해 반복될 수 있다.

즉, 최악의 경우에는 Brute-force와 비슷하게 많은 비교가 발생하므로 문자열 탐색 과정의 시간 복잡도는 `O(nm)`이 된다.

또한 알고리즘을 실행하기 전에 last occurrence 함수 `L`을 생성해야 한다. 이때 패턴의 길이를 `m`, 알파벳 집합의 크기를 `s`라고 하면 전처리 비용은 `O(m + s)`이다.

따라서 Boyer-Moore-Horspool 알고리즘의 전체 최악 시간 복잡도는 `O(nm + s)`이다.

### 이전에 다룬 KMP보다 더 느린 것 아닌가?

최악 시간 복잡도만 보면 BMH는 `O(nm + s)`이고, KMP는 `O(n + m)`이므로 KMP가 더 빠르고 안정적이다.

하지만 BMH의 worst-case는 일반적인 영어 텍스트에서 거의 발생하지 않는다. 실제로는 불일치가 발생할 때 패턴을 여러 칸씩 이동할 수 있기 때문에 평균적으로 `O(n/m)` 수준으로 매우 빠르게 동작한다.

따라서 이론적으로는 KMP가 더 좋지만, 실무에서는 BMH가 더 빠른 경우가 많고 구현도 훨씬 간단하다.

### 어디서 쓰이는가?
문자열을 찾는 알고리즘이기 때문에, 개인적으로는 검색 기능이나 `Ctrl + F`처럼 특정 문자열을 빠르게 찾는 기능에 많이 쓰일 것 같다고 생각했다. 결국 핵심은 긴 텍스트 안에서 내가 원하는 패턴 문자열이 존재하는지 비교하고 탐색하는 것이기 때문이다.

실제로 Boyer-Moore 계열 알고리즘은 문자열 검색 기능에서 자주 언급된다. 예를 들어 C++ 표준 라이브러리에는 `boyer_moore_horspool_searcher`가 존재하며, 이는 생성자에 지정한 패턴 시퀀스를 다른 시퀀스 안에서 검색하는 함수 객체로 사용된다. 즉, 단순히 이론적인 알고리즘이 아니라 실제 라이브러리에서도 문자열 또는 일반 시퀀스 검색을 위해 제공되는 방식이다.

또한 Boost 라이브러리에서도 Boyer-Moore-Horspool 검색 알고리즘을 제공한다. Boost 문서에서는 BMH가 Boyer-Moore 알고리즘의 변형이며, Boyer-Moore보다 내부 테이블에 필요한 공간을 줄인 방식이라고 설명한다. 대신 최악의 경우 성능은 더 나쁠 수 있지만, 실제 검색 과정에서는 간단하고 빠르게 사용할 수 있다는 장점이 있다. 

따라서 BMH 알고리즘은 `Ctrl + F` 같은 텍스트 검색, 문서 내부 문자열 탐색, 코드 편집기나 라이브러리의 substring search처럼 긴 문자열 안에서 특정 패턴을 찾는 상황에 활용될 수 있다. 특히 검색할 패턴을 미리 알고 있고, 그 패턴을 긴 텍스트에서 반복적으로 찾는 경우에는 last occurrence 정보를 전처리해두는 방식이 효과적으로 작동할 수 있다.
