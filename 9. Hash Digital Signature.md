
# Hash Function
해쉬 함수는 어떤 값 x를 입력 받아서 고정된 길이의 출력값 `H(x)` 를 만드는 함수이다.

해쉬 함수는 두 가지 성질을 가진다.

#### 1. Compression
입력의 길이가 얼마든지 될 수 있다. 짧은 문자열일 수도 있고, 긴 문자열일 수도 있는데, 항상 고정된 길이로 나온다.

#### 2. Ease of computation
입력 x가 주어졌을 때 `H(x)`를 계산하는 것은 쉬워야 한다.

# Hash Function & MAC

![](../images/Pasted%20image%2020260517175102.png)

일반 해시 함수는 message 를 Hash Function에 넣으면 고정 길이인 Hash value 가 나온다. 여기에는 비밀키가 없다.

반면 MAC 은 message 와 secret key를 더해서 MAC function 안에 넣어서 MAC value 가 나온다. MAC 은 메세지 뿐만 아니라 비밀키도 같이 사용한다. 그래서 키를 모르는 사람은 올바른 MAC 값을 만들 수 없다.

Hash Function 과 MAC 의 중요한 차이는 인증이 가능한가 아닌가 이다. Hash 같은 경우에는 무결성 확인이 가능하지만 누가 만들었는지 인증은 할 수 없다. 하지만 MAC 은 중간에 key를 사용하므로 무결성과 인증을 둘 다 할 수 있다.

# Hash Function의 목적
해시 함수 H는 가변 길이 데이터 M을 입력 받고 고정 길이 해시값 h를 만든다.

`h = H(M)` 

해시 함수의 가장 큰 목적은 Data integrity, 데이터 무결성이다.
예를 들어 어떤 파일을 다운로드 했을 때, 제공자가 SHA-256 해시값을 같이 올려두는 경우가 있다. 사용자는 파일을 받은 후 직접 해시를 계산해서 제공된 해시값과 비교한다.  같으면 파일이 변조되지 않았다고 판단할 수 있다.

그런데 암호학적인 해시 함수는 그냥 요약만 하면 안되고, 두 가지 조건을 만족해야 한다.

#### One way property 일방향성
첫 번째, 특정 해시값이 주어졌을 때, 그 해시 값을 만드는 입력을 찾기 어려워야 한다.

#### collision free property 충돌저항성
두 번재, 서로 다른 두 입력이 같은 해시값을 만들도록 찾기 어려워야 한다.

# Checking Integrity

![](../images/Pasted%20image%2020260517175639.png)

해시를 이용해서 무결성을 확인하는 방법은 간단하다. 먼저 원본 데이터의 해시 값을 저장해둔다. 예시에서는 previous digest이다. 그리고 나중에 데이터를 받는 사람은 현재 데이터의 해시값을 다시 계산한다. 이것을 Current digest 라고 하고, 두 값을 비교한다.

값이 같으면 데이터가 바뀌지 않은 것이고, 다르면 데이터가 변경된 것이다.

# Attack Against Hash Function

![](../images/Pasted%20image%2020260517175919.png)

해시 함수 공격은 크게 3가지로 나뉜다. 

첫 번째는 Preimage attack이다.
해시  h가 주어졌을 때, `H(x) = h` 가 되는 입력 x를 찾는 공격이다.

두 번째는 Second preimage attack이다.
어떤 원본 메세지 x가 주어졌을 때, `H(y) = H(x)` 가 되는 다른 메세지 y를 찾는 공격이다.

세 번째는 Collison attack 이다.
공격자가 아무 두 메세지 x, y를 골라서 `x ≠ y`인데 `H(x) = H(y)`가 되게 찾는 공격이다.

이 중에서는 collision attack이 상대적으로 더 쉽다.
