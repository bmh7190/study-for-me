
사설망에서는 사설망 전용 IP 주소를 사용하지만, 이러한 주소는 인터넷에서 유효하지 않기 때문에 **외부 네트워크와 통신하기 위해서는 공인 IP로 변환하는 과정이 필요하다.** 이 역할을 수행하는 것이 바로 **NAT(Network Address Translation)** 이다.

![](../../images/Pasted%20image%2020251207163224.png)

예를 들어 내부 장비가 `10.0.0.1` 과 같은 사설 IP를 갖고 있다고 하자. 이 주소로 인터넷에 요청을 보내면 응답이 돌아올 수 없으므로, 외부로 나가기 전에 **라우터가 사설 IP를 공인 IP로 교체**해준다. 이때 라우터는 단순히 주소만 바꾸는 것이 아니라, 어떤 내부 주소가 어떤 공인 IP·포트를 사용했는지를 **NAT 변환 테이블에 기록**한다.

- 내부 IP: 10.0.0.1
    
- 내부 Port: 3000
    
- 외부에서 사용할 Public IP: 203.x.x.x
    
- 변환된 Port: 50000
    

같은 식으로 매핑해 저장하는 방식이다.

이제 외부 서버는 요청을 보낸 장치를 `203.x.x.x:50000` 으로 인식하게 되고, 응답 역시 그 주소로 보내게 된다. 라우터는 응답을 받으면 NAT 테이블을 확인하여 이를 원래 내부 주소(`10.0.0.1:3000`)로 되돌려 전달한다.

결국 NAT는 **사설망 장비가 공인 IP 없이도 외부와 통신할 수 있도록 중간에서 주소 변환과 연결 매핑을 수행해주는 메커니즘**이라고 이해하면 된다.
